## Features

### The Invisible Cloak â€” Privacy & Anonymity

Sometimes the best agent is the one nobody sees.

#### Tor Integration (Built-in)
- **One-Click Tor Activation** â€” Route all traffic through the Tor network
- **DNS Leak Prevention** â€” Uses `socks5h://` protocol for DNS resolution through Tor
- **No External Dependencies** â€” Just have Tor running on port 9050

#### Anti-Fingerprinting Arsenal
- **Canvas Fingerprint Randomization** â€” Adds noise to canvas data extraction
- **WebGL Fingerprint Spoofing** â€” Reports generic Intel GPU info
- **Screen Resolution Masking** â€” Reports standard 1920x1080 resolution
- **Audio Fingerprint Protection** â€” Prevents audio-based tracking
- **Battery API Removal** â€” Blocks battery status tracking

#### Tracker Annihilation
Blocks **50+ known tracking domains** including:
- Google Analytics, Tag Manager, AdServices
- Facebook Pixel, Connect
- Twitter/X Analytics
- TikTok Analytics
- Hotjar, Mixpanel, Amplitude, Segment
- Criteo, Outbrain, Taboola
- And many more...

#### Privacy Headers
- **Do Not Track (DNT)** â€” Sends DNT header with every request
- **Global Privacy Control (GPC)** â€” The new standard for privacy signals
- **User Agent Spoofing** â€” Blend in with Firefox ESR users
- **HTTPS Upgrade** â€” Automatic upgrade from HTTP to HTTPS
- **WebRTC IP Leak Prevention** â€” No accidental IP exposure

## Installation

### System Requirements

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| **Node.js** | 18.x | 20.x or later |
| **npm** | 8.x | 10.x or later |
| **RAM** | 4 GB | 8 GB |
| **Disk Space** | 500 MB | 1 GB |
| **OS** | Windows 10, macOS 10.15, Ubuntu 20.04 | Latest versions |

### Step 1: Clone the Repository

```bash
# Clone the SANDIEGO repository
git clone https://github.com/thumpersecure/Spin.git

# Navigate into the project directory
cd Spin
```

### Step 2: Install Dependencies

```bash
# Install all required packages
npm install

# This will automatically:
# - Install Electron 35.7.5
# - Install electron-builder for packaging
# - Install electron-store for settings persistence
# - Configure app dependencies
```

**Troubleshooting Installation:**

If you encounter permission errors on Linux/macOS:
```bash
sudo chown -R $(whoami) ~/.npm
npm install
```

If you encounter network errors:
```bash
npm install --registry https://registry.npmjs.org/
```

### Step 3: Start the Browser

```bash
# Production mode (recommended)
npm start

# Development mode (with DevTools)
npm run dev
```

## Tor Setup (Optional but Recommended)

For full anonymity, you'll need the Tor daemon running on your system.

### Linux (Debian/Ubuntu)

```bash
# Install Tor
sudo apt update
sudo apt install tor

# Start Tor service
sudo systemctl start tor

# Enable Tor to start on boot (optional)
sudo systemctl enable tor

# Verify Tor is running on port 9050
sudo netstat -tlnp | grep 9050
```

### Linux (Fedora/RHEL)

```bash
# Install Tor
sudo dnf install tor

# Start Tor service
sudo systemctl start tor
```

### Linux (Arch)

```bash
# Install Tor
sudo pacman -S tor

# Start Tor service
sudo systemctl start tor
```

### macOS

```bash
# Install via Homebrew
brew install tor

# Start Tor
brew services start tor

# Or run manually
tor
```

### Windows

1. Download the **Tor Expert Bundle** from [torproject.org](https://www.torproject.org/download/tor/)
2. Extract to a folder (e.g., `C:\Tor`)
3. Open Command Prompt as Administrator
4. Navigate to the Tor folder and run:
   ```cmd
   tor.exe
   ```
5. Keep the window open while using SANDIEGO

**Verify Tor is Running:**
```bash
# Should show something listening on port 9050
# Linux/macOS
lsof -i :9050

# Windows (PowerShell)
netstat -an | findstr 9050
```

## Usage

### First Launch

1. **Start SANDIEGO** â€” Run `npm start` or launch the built application
2. **Enable Privacy Features** â€” Click the shield icon and configure your privacy settings
3. **Connect to Tor** (optional) â€” Toggle "Tor Proxy" in Privacy settings (ensure Tor is running)
4. **Browse Securely** â€” Use the search bar or OSINT tools panel

### Privacy Dashboard

Access via the shield icon in the toolbar:

| Setting | Description | Default |
|---------|-------------|---------|
| **Tor Proxy** | Route traffic through Tor network | Off |
| **Block Trackers** | Block 50+ known tracking domains | On |
| **Block Fingerprinting** | Randomize browser fingerprint | On |
| **Block Third-Party Cookies** | Prevent cross-site tracking | On |
| **Block WebRTC** | Prevent IP leaks via WebRTC | On |
| **Spoof User Agent** | Use Firefox user agent | On |
| **Do Not Track** | Send DNT/GPC headers | On |
| **HTTPS Upgrade** | Auto-upgrade HTTP to HTTPS | On |
| **Clear on Exit** | Delete all data when closing | Off |

### Using Phone Intelligence

1. Click the **Extensions** button (puzzle piece icon)
2. Select the **Phone Intel** tab
3. Enter a phone number
4. Select the country
5. Choose your search engine (DuckDuckGo recommended for privacy)
6. Click **Generate Format Variations** to see all formats
7. Click **Smart OSINT Search** to search all formats at once

### Keyboard Shortcuts

| `Ctrl/Cmd + -` | Zoom out |

## Security Features

### Electron Security Hardening

SANDIEGO implements all recommended Electron security practices:

- **Context Isolation** â€” Renderer process isolated from Node.js
- **Sandbox Mode** â€” Enabled for all browser views
- **Node Integration Disabled** â€” No direct Node.js access from renderer
- **Web Security Enabled** â€” Same-origin policy enforced
- **Remote Module Disabled** â€” All remote module events blocked
- **File Protocol Blocked** â€” No `file://` URL navigation from web content
- **JavaScript URL Blocked** â€” No `javascript:` URL execution

### Input Validation

- Phone numbers limited to 30 characters
- Search URLs validated against trusted hosts only
- IPC channel whitelist for event listeners
- URL processing with protocol validation

### Privacy by Design

- No telemetry or analytics
- No external API calls (except user-initiated searches)
- All settings stored locally with electron-store
- Optional auto-clear on exit

## Security Considerations

### For Investigators

- **Use Tor** for sensitive investigations
- **Enable all privacy features** for maximum protection
- **Clear data on exit** for sensitive sessions
- **Verify HTTPS** before entering any credentials
- **Use multiple sources** â€” never trust a single result

### Limitations

- **Tor is not invincible** â€” Sophisticated adversaries may still track you
- **JavaScript can detect Tor** â€” Some sites block Tor exit nodes
- **WebRTC is disabled** â€” Video conferencing may not work
- **Fingerprinting protection may break sites** â€” Some features rely on accurate data
